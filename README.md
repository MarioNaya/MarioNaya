# Mario Naya

Software Engineer · Agentic Testing Systems · [Hiberus](https://www.hiberus.com)

Construyo sistemas de testing automatizado con agentes LLM para proyectos de gran escala. Trabajo en la intersección entre ingeniería de software tradicional y arquitectura de sistemas agénticos: memoria persistente entre sesiones, orquestación de agentes especializados, y patrones de prompt engineering aplicados a flujos reales de desarrollo.

## En lo que estoy trabajando

**[choreon-framework](https://github.com/MarioNaya/choreon-framework)** — Framework personal en desarrollo para Spec-Driven Development con agentes de IA. Explora una arquitectura coreografiada (sin orquestador central) basada en sistemas desacoplados con contratos tipados estrictos. En fase temprana de construcción.

**[task-aware-pointer](https://github.com/MarioNaya/task-aware-pointer)** — Implementación de referencia del patrón de punteros task-aware descrito en mi [artículo en dev.to](https://dev.to/marionaya). Un mecanismo de memoria que desplaza la decisión de selección de contexto del arranque al cierre de sesión, permitiendo que cada sesión arranque con convenciones pre-seleccionadas para su próxima tarea.
